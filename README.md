# UVM_AI_DEMO

## 项目简介

本项目是一个基于 UVM（Universal Verification Methodology）的验证框架，旨在规范化和简化芯片验证工作流程。

## EDA 工具

| 工具 | 版本 |
|------|------|
| Synopsys VCS | 2020 |

所有编译脚本、仿真脚本及环境配置均基于 **VCS 2020** 版本生成，不保证对其他版本的兼容性。

## 目录结构

```
UVM_AI_DEMO/
├── design/      # DUT（被测设计）RTL 代码
├── verif/       # Testbench、VIP、验证激励
└── scripts/     # 编译、仿真、回归等自动化脚本
```

## 基本要求

### 1. 代码组织
- `design/`：存放 **RTL**（被测设计）代码
- `verif/`：存放 **VIP**、Testbench、Sequence、Test 等验证组件
- `scripts/`：存放所有自动化脚本

### 2. 验证流程封装
- 封装**编译**步骤，统一管理文件列表与编译选项
- 封装**回归**（Regression）步骤，支持批量用例执行与结果汇总

### 3. 脚本化
- 脚本语言：**Makefile**、**Python3**、**Shell**
- 提供脚本简化验证操作，降低手动操作成本
- 常用操作（编译、仿真、回归、报告生成）均可通过脚本一键完成

### 4. 代码规范
- 代码须有**适当注释**，说明模块功能、接口含义及关键逻辑
- 保持较强的**可读性**，命名清晰，结构整洁
